02 · Algorithms

六种分类算法：原理、优点与局限

每张卡片同时展示算法知识与本次真实测试结果，方便把理论特征和实际性能联系起来。

01

逻辑回归

可解释的线性概率基准

AUC 0.9950 F1 0.9718 AP 0.9969

原理：计算特征的线性组合，再通过 Sigmoid 函数映射为 0～1 的概率。

优点：训练快、输出概率、参数含义清晰，适合作为分类任务的基准模型。

局限：主要学习线性决策边界，复杂非线性关系通常需要特征工程。

本次配置：StandardScaler + class_weight='balanced'

02

决策树

直观的非线性规则模型

AUC 0.9180 F1 0.9209 AP 0.9231

原理：按能最大程度降低节点不纯度的特征和切分点递归划分样本。

优点：规则直观、可解释，能够学习非线性关系，且无须特征标准化。

局限：单棵树容易过拟合，对训练样本变化较敏感。

本次配置：max_depth=4, min_samples_leaf=5

03

随机森林

多棵随机决策树集成

AUC 0.9927 F1 0.9510 AP 0.9958

原理：通过 Bootstrap 抽样训练多棵树，并在随机特征子集上分裂，最终投票。

优点：准确稳定、非线性能力强、抗异常值，并可评估特征重要性。

局限：模型体积和计算成本高于单棵树，整体规则不易直接解释。

本次配置：300 trees + class_weight='balanced'

04

支持向量机（RBF）

最大间隔非线性分类器

AUC 0.9944 F1 0.9718 AP 0.9966

原理：寻找分类间隔最大的边界，RBF 核将样本映射到可处理非线性关系的空间。

优点：适合中小规模和高维数据，复杂边界下通常表现良好。

局限：大样本训练较慢，参数和预测结果不如线性模型容易解释。

本次配置：StandardScaler + RBF + probability=True

05

K近邻

邻近样本多数投票

AUC 0.9901 F1 0.9655 AP 0.9939

原理：寻找距离新样本最近的 K 个训练样本，根据距离加权投票确定类别。

优点：原理简单，不预设线性形式，能够拟合非线性分类边界。

局限：预测较慢，对特征尺度、无关特征和 K 值敏感。

本次配置：StandardScaler + K=7 + distance weights

06

高斯朴素贝叶斯

基于条件概率的快速模型

AUC 0.9858 F1 0.9371 AP 0.9915

原理：利用贝叶斯公式计算后验概率，并假设给定类别后各特征条件独立且服从高斯分布。

优点：训练和预测很快，在小样本、高维任务中可作为有效基准。

局限：特征独立与高斯分布假设较强，相关特征较多时可能限制效果。

本次配置：StandardScaler + GaussianNB

06 · Metrics

评价指标完整解释

分类模型不存在对所有任务都最好的单一指标。应根据类别分布、误报/漏报成本及是否关注概率质量进行组合判断。

1. Accuracy 准确率

(TP + TN) / (TP + TN + FP + FN)

全部预测中正确的比例，直观但在类别严重不平衡时可能产生误导。

2. Precision 精确率

TP / (TP + FP)

预测为正的样本中实际为正的比例。越高表示误报越少。

3. Recall 召回率 / TPR

TP / (TP + FN)

全部真实正类中被成功找出的比例。越高表示漏报越少。

4. Specificity 特异度

TN / (TN + FP) = 1 − FPR

全部真实负类中被正确识别为负类的比例，反映排除负类的能力。

5. F1 分数

2 × Precision × Recall / (Precision + Recall)

Precision 与 Recall 的调和平均，适合同等重视误报和漏报的任务，但不考虑 TN。

6. ROC 曲线

横轴 FPR = FP/(FP+TN)；纵轴 TPR = TP/(TP+FN)

改变分类阈值得到一系列 FPR/TPR 点。曲线越靠近左上角，分类能力越好。

7. ROC-AUC

Area Under the ROC Curve ∈ [0, 1]

衡量跨阈值的整体排序能力；可理解为随机正样本得分高于随机负样本的概率。它不等于概率校准质量。

8. PR 曲线与 AP

横轴 Recall；纵轴 Precision

类别极不平衡时通常比 ROC 更敏感。AP 汇总不同召回率下的 Precision，越高越好。

9. Log Loss 交叉熵

−mean[y·log(p) + (1−y)·log(1−p)]

衡量预测概率质量，越低越好；对“非常自信但错误”的预测惩罚很大。

10. 阈值与业务成本

默认分类阈值 = 0.5

降低阈值通常提高 Recall 但增加 FP；提高阈值可能提高 Precision 但增加 FN。最终阈值应由业务成本决定。

07 · Conclusion

结论与选择建议

本次结果：逻辑回归 获得最高 ROC-AUC（0.9950）。但模型选择不应只看 AUC，还应结合 F1、AP、Log Loss、误报/漏报数量、解释需求和计算成本。
  • 需要概率与解释：
    优先观察逻辑回归，它是稳定且容易解释的基准。
  • 重视综合预测性能：
    重点比较随机森林和 SVM 的 AUC、AP 与错误类型。
  • 需要可读规则：
    决策树最直观，但要限制深度并警惕过拟合。
  • 正类非常稀少：
    重点查看 PR 曲线、AP、Precision 和 Recall，而非只看 Accuracy。

方法限制：本报告采用固定的单次分层划分，适合教学演示和初步比较。若用于正式建模，应进一步采用分层交叉验证、超参数搜索、概率校准和独立外部测试集。
